HR Data Classifier
Background Analysis

HR Data Quality Analyzer is a python Flask Based portal to assess the quality of data in the core HR system. Using machine learning classifiers, this tool evaluated data accuracy and completeness, identifying specific groups needing targeted communication and training. This approach ensured that data quality issues were systematically addressed and that corrective actions were prioritized based on data integrity insights.

Tools Used

The HR Data Classifier project incorporated a range of machine learning techniques to evaluate and improve data quality. Key models used included K-Nearest Neighbors (KNN), Logistic Regression, Naive Bayes, Decision Trees, and a Voting Classifier to ensure robust predictions. Principal Component Analysis (PCA) was applied to reduce dimensionality and enhance visualization. Together, these tools provided a comprehensive analysis of data patterns, highlighting areas where targeted communication and training could drive improvements

A3 Thinking - Lean Six Sigma
Background Analysis

The payroll process was experiencing inefficiencies, with delays and errors impacting employee satisfaction and creating compliance risks. Analysis revealed bottlenecks due to redundant manual steps, data inconsistencies, and approval delays, which collectively increased the cycle time and error rate.

Tools Used

Using Value Stream Mapping as a primary tool, we mapped the entire payroll process to identify non-value-added steps and pinpoint areas of waste. This analysis highlighted key issues such as lack of standardization, manual data entry errors, and approval bottlenecks, which were all contributing to inefficiencies.

JIRA Dashboard
JIRA Dashboards: Test Management for Global HR System Implementation

As part of the global rollout of HR systems across over 80 countries, effective test management was essential to ensure smooth implementation and minimize issues during deployment. I developed a series of JIRA Dashboards to track test progress, manage issues, and ensure that all test cases were executed and resolved on time. The dashboards were designed to provide real-time visibility into test activities, including:

  • Test Execution Progress: Monitored the status of test cycles and individual test cases, tracking completion rates and open issues.
  • Issue Management: Highlighted critical defects and their impact on the overall project timeline, enabling quick resolution.
  • Test Coverage: Ensured that all HR system modules were thoroughly tested across different regions and user groups.
  • Global Tracking: Allowed for tracking and comparison of test progress across countries, ensuring consistency in test results and process adherence.
